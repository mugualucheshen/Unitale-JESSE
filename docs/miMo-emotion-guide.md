# MiMo TTS 情绪控制 — 最丰富用法指南

> **场景**:为 Unitale 有声书工具的配音功能,梳理如何用最丰富的方式控制小米 TTS 的情绪。
> **核心结论**:三种方式可叠加在同一请求中,层层递进控制(整体→段落→句子→动作)。

## 三种方式一览

| 方式 | 位置 | 颗粒度 | 适用场景 |
|---|---|---|---|
| **自然语言导演模式** | `role: user` | 整体(全段) | 锁定角色、场景、风格基调 |
| **`(风格)` 括号标签** | `role: assistant` 开头 | 段落/句 | 基础/复合情绪、整体语调、音色定位 |
| **`[音频]` 方括号标签** | `role: assistant` 任意位置 | 词/动作 | 哭/笑/呼吸/颤抖等细粒度情绪动作 |

**调用规则**(摘自小米文档):
- 合成文本必须在 `role: assistant`
- `role: user` 用来注入风格指令(`voicedesign` 模型必填,`voiceclone` 模型必须为空)
- **所以在 voiceclone 模式下,情绪只能走 assistant 文本的 `(风格)` + `[音频]` 标签**

---

## 一、自然语言导演模式(写在 `role: user`)

### 基础写法(整体风格)

一句话直接描述:

> "Bright, bouncy, slightly sing-song tone — like you are bursting with good news you can barely hold in."

### 富写法(导演模式,角色 + 场景 + 指导)

```
【角色】30岁出头的女外卖员,疲惫但嘴硬,带点自嘲,说话简短不啰嗦
【场景】凌晨2点送完最后一单,在电动车上,对着副驾上的师傅说话
【指导】语速偏慢,声音略带沙哑,气息感重;开头极度疲惫,中段有点小抱怨,末尾转为释然
```

**可控制维度**:
- 身份 / 性格 / 外形 / 说话习惯
- 时间 / 地点 / 对象 / 情绪位置
- 语速 / 气息 / 停顿 / 重音 / 共鸣 / 音色质感 / 情绪起伏

---

## 二、`(风格)` 括号标签(写在 assistant 文本开头)

### 支持的中文标签(摘自官方)

| 类别 | 标签 |
|---|---|
| 基础情绪 | 开心 / 悲伤 / 愤怒 / 恐惧 / 惊讶 / 兴奋 / 委屈 / 平静 / 冷漠 |
| 复合情绪 | 怅然 / 欣慰 / 无奈 / 愧疚 / 释然 / 嫉妒 / 厌倦 / 忐忑 / 动情 |
| 整体语调 | 温柔 / 高冷 / 活泼 / 严肃 / 慵懒 / 俏皮 / 深沉 / 干练 / 凌厉 |
| 音色定位 | 磁性 / 醇厚 / 清亮 / 空灵 / 稚嫩 / 苍老 / 甜美 / 沙哑 / 醇雅 |
| 人设腔调 | 夹子音 / 御姐音 / 正太音 / 大叔音 / 台湾腔 |
| 方言 | 东北话 / 四川话 / 河南话 / 粤语 |
| 角色 | 孙悟空 / 林黛玉 |
| 唱歌 | 唱歌 / sing |

**括号兼容**:半角 `()`、全角 `（）`、`[]` 都接受。

### 富写法(组合多个标签)

```
(温柔 沙哑) 多年不见,你怎么老成这样了。
(怅然 释然) 都过去了,就别再提。
(磁性 磁性 磁性) 重要的事情说三遍。
```

> 注:重复相同标签可强化效果(小米的"加重"机制)。

## 三、`[音频]` 方括号标签(写在 assistant 文本任意位置)

### 支持的音频标签(摘自官方)

| 类别 | 标签 |
|---|---|
| 呼吸节奏 | 吸气 / 深呼吸 / 叹气 / 长叹一口气 / 喘息 / 屏息 |
| 情绪动作 | 紧张 / 害怕 / 激动 / 疲惫 / 委屈 / 撒娇 / 心虚 / 震惊 / 不耐烦 |
| 声音特征 | 颤抖 / 声音颤抖 / 变调 / 破音 / 鼻音 / 气声 / 沙哑 |
| 哭笑表达 | 笑 / 轻笑 / 大笑 / 冷笑 / 抽泣 / 呜咽 / 哽咽 / 嚎啕大哭 |

### 富写法(穿插在正文中)

```
[长叹一口气]唉……算了,跟你说不清。[小声嘟囔]反正我说了算。
[紧张]他、他不会真来了吧?[屏息]别出声。
[破音]不——!你怎么能这样!
[哽咽]对不起……我、我不是故意的……
```

**位置自由**:可放在句首、句中、句尾;同一句话可叠加多个标签,触发多个音效。

## 四、最丰富组合:三层叠加(完整 cURL)

### 场景:师徒深夜对话

**台词正文**(30秒片段):
> 师傅……到地方了叫我一声……我先眯一会儿,这班加得我魂儿都要散了。

**情绪设计**:
- user 消息:角色+场景+指导(整体锁基调)
- assistant 开头 `(极其疲惫 沙哑)`:段落情绪
- 文中 `[长叹一口气]`:动作音效
- 中段 `(委屈)`:情绪转折
- 末尾 `[小声嘟囔]`:语气细节

**完整请求**:

```bash
curl --location --request POST 'https://api.xiaomimimo.com/v1/chat/completions' \
  --header "api-key: $MIMO_API_KEY" \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "model": "mimo-v2.5-tts-voiceclone",
    "messages": [
      {
        "role": "user",
        "content": "【角色】30岁出头的女外卖员,疲惫但嘴硬,带点自嘲\n【场景】凌晨2点送完最后一单,坐在电动车上,对着坐在后座的师傅说话\n【指导】语速偏慢,声音略带沙哑,气息感重;开头极度疲惫,中段有点小抱怨,末尾转为释然和自嘲"
      },
      {
        "role": "assistant",
        "content": "(极其疲惫 沙哑)[长叹一口气]师傅……到地方了叫我一声……(委屈 嘟囔)[小声嘟囔]我先眯一会儿,(释然)这班加得我魂儿都要散了。"
      }
    ],
    "audio": {
      "format": "wav",
      "voice": "data:audio/wav;base64,XXX"
    }
  }'
```

**预期效果层次**:
1. **整体**(user):疲惫女外卖员、凌晨电动车场景
2. **段落**(assistant 开头):沙哑疲惫的基调
3. **动作**(`[长叹一口气]`):真实叹气音效
4. **转折**(`(委屈 嘟囔)`):情绪从疲惫到抱怨
5. **收束**(`(释然)` + `[小声嘟囔]`):自嘲收尾

## 五、其他富写法示例

### 例 1:电话通话(模拟滤波器场景)

```json
{
  "model": "mimo-v2.5-tts-voiceclone",
  "messages": [
    {"role": "user", "content": ""},
    {"role": "assistant", "content": "(电话音 沙哑)[紧张]喂?听得见吗?[屏息]信号不太好,我再说一遍——(急促)我到楼下了,你下来一趟!"}
  ],
  "audio": {"format": "wav", "voice": "data:audio/wav;base64,XXX"}
}
```

### 例 2:战斗场景配音

```json
{
  "model": "mimo-v2.5-tts-voiceclone",
  "messages": [
    {"role": "user", "content": ""},
    {"role": "assistant", "content": "(凌厉 杀气)[冷喝]挡我者死![破音]哈——!(喘气)没想到……你竟然还藏了一手。" }
  ],
  "audio": {"format": "wav", "voice": "data:audio/wav;base64,XXX"}
}
```

### 例 3:哭戏独白

```json
{
  "model": "mimo-v2.5-tts-voiceclone",
  "messages": [
    {"role": "user", "content": ""},
    {"role": "assistant", "content": "(哽咽 颤抖)[抽泣]妈……妈你别走……[呜咽]我还没来得及……带你去看看海……(哽咽 破音)[嚎啕大哭]妈——!"}
  ],
  "audio": {"format": "wav", "voice": "data:audio/wav;base64,XXX"}
}
```

### 例 4:唱歌模式

```json
{
  "model": "mimo-v2.5-tts",
  "messages": [
    {"role": "user", "content": "舒缓、温暖的女声,民谣风"},
    {"role": "assistant", "content": "(唱歌)月亮挂在天上,照着谁回家的路……"}
  ],
  "audio": {"format": "wav", "voice": "Chloe"}
}
```

> 注:唱歌模式必须在文本最开头加 `(唱歌)`,中文歌词效果更优。

## 六、迁移到本项目的方案(待你拍板)

### 当前 bug

我们的 `generateLineAudio` 在 voiceclone 模式下:
- ✓ 计算了情绪 vector → styleDesc(自然语言)
- ✗ 但 `buildVoiceClonePayload` 的 user 强制为空(per spec),styleDesc 没用上
- ✗ assistant 文本也只放纯台词,没有 `(风格)` 标签

→ **结果**:脚本台本上的情绪完全没传到 MiMo,合成出来永远是中性。

### 修复方案:把情绪翻译成 `(风格)` 标签塞进 assistant 文本

```js
// 之前
content: line.text  // "(开心)台词正文" 没有这种处理

// 之后
content: line.emotion
  ? `(${emotionName} ${intensityWord})${line.text}`  // "(开心 强烈)台词正文"
  : line.text
```

**强度词映射**(基于现有 0.3/0.4/0.75 阈值):
- `> 0.75` → `强烈`
- `> 0.4` → `明显`
- `≤ 0.4` → `轻微`

**好处**:
- 不破坏现有数据结构(`line.emotion` 字段不变)
- 用户自定义情绪(非系统 8 种)可走默认标签 `(自然)` 或直接放行
- 后续可扩展:角色面板让用户填额外 `(音频标签)` 字段

### 进阶(后续再做)

- 角色面板加"情绪标签"输入框,允许用户填 `(温柔 沙哑)` 等复合标签
- 自动把系统情绪映射到对应小米标签(高兴 → 开心, 生气 → 愤怒)
- LLM 拆分剧本时直接生成小米格式的标签
